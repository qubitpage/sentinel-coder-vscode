# Support, donations, and community

Sentinel Coder One Studio is an open-source project maintained by QubitPage Research.

- Repository: https://github.com/qubitpage/sentinel-coder-vscode
- Issues and feature requests: https://github.com/qubitpage/sentinel-coder-vscode/issues
- Contribution guide: ../CONTRIBUTING.md
- Donation link: https://www.paypal.com/donate/?hosted_button_id=97VNNYCB3HWMS

If Sentinel Coder One Studio helps your work and you want to support frequent updates, any amount is useful to sustain development. Donations help keep the project open source, maintained, tested, and improving across VS Code Desktop, VS Code Web, model providers, media generation, and enterprise workflows.

![PayPal donation QR](assets/paypal-donation-qr.png)

## What donations support

Donations help cover:

1. Marketplace publishing and compatibility maintenance.
2. Test runs across VS Code Desktop and VS Code Web.
3. Provider integration work for Azure OpenAI, OpenAI-compatible APIs, Groq, OpenRouter, Anthropic, Ollama, and future providers.
4. Security reviews, secret scans, package-hardening, and webview-hardening work.
5. Documentation, examples, onboarding guides, and contribution support.
6. Continued open-source development instead of locking core workflows behind a closed product.

## Contribution principles

- Keep API keys, tokens, customer files, and private endpoints out of public issues and pull requests.
- Prefer small, reviewable pull requests.
- Include test evidence and screenshots for UI changes.
- For new providers, document required environment variables/settings and supported operations.
- For web compatibility, explain which features work natively in the browser and which require a remote bridge or desktop host.

## Community requests we welcome

- Provider/model discovery improvements.
- Agentic Profile templates for cost, speed, reasoning, privacy, and enterprise review.
- VS Code Web and remote bridge compatibility reports.
- Documentation fixes and tutorials.
- Security-hardening suggestions.
- Accessibility improvements.
- Studio/media workflow examples.
